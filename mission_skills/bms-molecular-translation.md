## Competition Details

| Field | Value |
|---|---|
| Competition | [BMS Molecular Translation](https://www.kaggle.com/competitions/bms-molecular-translation) |
| Task | Translate chemical structure images into IUPAC InChI (International Chemical Identifier) strings |
| Input | PNG images of molecular structure diagrams |
| Target | InChI string, e.g. `InChI=1S/C9H10O2/c1-6...` |
| Evaluation | Mean Levenshtein distance between predicted and ground-truth InChI strings (lower is better) |

## Architecture Comparison

| Aspect | Part I | Part II |
|---|---|---|
| **Encoder** | EfficientNet-B5 (timm) | ResNet-18 (torchvision) |
| **Encoder output** | `(B, H, W, 2048)` → flattened to `(B, H×W, 2048)` | `(B, 8, 8, 512)` → flattened to `(B, 64, 512)` |
| **Encoder training** | Frozen (pretrained weights loaded) | Fully trainable (no frozen params) |
| **Decoder** | 2-layer LSTMCell stack | Single LSTMCell |
| **Attention activation** | ReLU | tanh |
| **Context gating** | Sigmoid gate (`f_beta`) | None |
| **Tokenization** | Space-split InChI tokens | Character-level (41 chars) |
| **Vocab type** | InChI layer/component tokens | Individual characters |
| **Vocab size** | Pre-fitted tokenizer (larger) | 41 (`stoi` hardcoded) |
| **Max decode length** | 275 (early stop at `<eos>`) | 202 (fixed, post-processed) |
| **Image transforms** | albumentations + cv2 | torchvision + PIL |
| **Orientation fix** | Portrait→landscape (transpose+vflip) | None |
| **Inference batch size** | 256 | 300 |
| **Submission InChI prefix** | Prepended manually (`"InChI=1S/"`) | Generated by model (full string) |

## Part I Pipeline

### Step 1 — Tokenizer (Space-Split InChI Tokens)

```python
class Tokenizer:
    def fit_on_texts(self, texts):
        vocab = sorted(set(token for text in texts for token in text.split(' ')))
        vocab += ['<sos>', '<eos>', '<pad>']
        self.stoi = {s: i for i, s in enumerate(vocab)}
        self.itos = {i: s for s, i in self.stoi.items()}

    def predict_caption(self, sequence):
        caption = ''
        for i in sequence:
            if i == self.stoi['<eos>'] or i == self.stoi['<pad>']:
                break
            caption += self.itos[i]
        return caption

tokenizer = torch.load('tokenizer2.pth')
```

- Tokens are space-delimited InChI sublayers (e.g. `C9H10O2`, `c1-6`, `h1-5H`) — larger vocab than character-level
- Pre-fitted on training InChI strings and saved to disk; loaded at inference time

### Step 3 — Encoder (EfficientNet-B5, Frozen)

```python
class Encoder(nn.Module):
    def __init__(self, model_name='tf_efficientnet_b5', pretrained=False):
        self.cnn = timm.create_model(model_name, pretrained=pretrained)

    def forward(self, x):
        features = self.cnn.forward_features(x)        # (B, 2048, H, W) spatial map
        features = features.permute(0, 2, 3, 1)        # (B, H, W, 2048)
        return features
```

- `forward_features()` returns the final conv feature map before global pooling
- Permuted to `(B, H, W, C)` then flattened to `(B, H×W, C)` in decoder as the spatial attention pixel set
- Encoder weights loaded from saved state dict; `pretrained=False` skips ImageNet download

### Step 4 — Attention (Additive, ReLU activation)

```python
class Attention(nn.Module):
    def forward(self, encoder_out, decoder_hidden):
        att1  = self.encoder_att(encoder_out)          # (B, num_pixels, att_dim)
        att2  = self.decoder_att(decoder_hidden)       # (B, att_dim)
        att   = self.full_att(F.relu(att1 + att2.unsqueeze(1))).squeeze(2)
        alpha = F.softmax(att, dim=1)
        awe   = (encoder_out * alpha.unsqueeze(2)).sum(dim=1)
        return awe, alpha
```

- ReLU on the summed projections before scoring
- `alpha`: softmax attention weights over `H×W` spatial pixels

### Step 5 — Decoder (2-Layer LSTM with Gated Attention)

```python
class DecoderWithAttention(nn.Module):
    def __init__(self, ..., num_layers=2):
        self.decode_step = nn.ModuleList([
            LSTMCell(embed_dim + encoder_dim if layer == 0 else embed_dim, embed_dim)
            for layer in range(num_layers)
        ])
        self.f_beta = nn.Linear(decoder_dim, encoder_dim)  # sigmoid gate on context
        self.fc     = nn.Linear(decoder_dim, vocab_size)
```

**Per-step greedy decoding:**
```python
for t in range(max_len):
    awe, alpha = self.attention(encoder_out, h[-1])    # attend from top LSTM layer
    gate  = self.sigmoid(self.f_beta(h[-1]))           # soft gate (0–1) per channel
    awe   = gate * awe                                 # gated context vector
    input = torch.cat([embeddings, awe], dim=1)

    for j, rnn in enumerate(self.decode_step):         # propagate through LSTM stack
        at_h, at_c = rnn(input, (h[j], c[j]))
        input = self.dropout(at_h)
        h[j], c[j] = at_h, at_c

    preds = self.fc(self.dropout(h[-1]))
    end_condition |= (torch.argmax(preds, -1) == tokenizer.stoi['<eos>'])
    if end_condition.sum() == batch_size:
        break                                          # early stop when all done
    embeddings = self.embedding(torch.argmax(preds, -1))
```

### Step 3 — Encoder (ResNet-18, Fully Trainable)

```python
class EncoderCNNtrain18(nn.Module):
    def __init__(self):
        resnet = torchvision.models.resnet18()
        # NOTE: param.requires_grad_(False) is commented out — encoder is trainable
        modules = list(resnet.children())[:-2]   # remove AvgPool and FC
        self.resnet = nn.Sequential(*modules)

    def forward(self, images):
        features = self.resnet(images)                                    # (B, 512, 8, 8)
        features = features.permute(0, 2, 3, 1)                           # (B, 8, 8, 512)
        features = features.view(features.size(0), -1, features.size(-1)) # (B, 64, 512)
        return features
```

- Final two layers (global average pool + FC) removed, preserving the `8×8` spatial feature grid
- 256×256 input → `8×8` spatial grid → **64 attention pixels** per image (vs. variable H×W in Part I)
- Encoder is **fully trainable** — parameter freeze is commented out; gradient flows through the backbone
- `encoder_dim=512` (vs. `enc_size=2048` in Part I)

### Step 4 — Attention (Additive, tanh activation)

```python
class Attention(nn.Module):
    def forward(self, features, hidden_state):
        u_hs = self.U(features)                                   # (B, 64, attention_dim)
        w_ah = self.W(hidden_state)                               # (B, attention_dim)
        combined_states = torch.tanh(u_hs + w_ah.unsqueeze(1))   # tanh (vs. ReLU in Part I)
        attention_scores = self.A(combined_states).squeeze(2)     # (B, 64)
        alpha = F.softmax(attention_scores, dim=1)                # (B, 64)
        attention_weights = (features * alpha.unsqueeze(2)).sum(dim=1)  # (B, 512)
        return alpha, attention_weights
```

- Uses **tanh** instead of ReLU for the combined projection — tanh is zero-centered and saturates symmetrically
- No sigmoid gate on the context vector (unlike Part I's `f_beta`)
- Fixed 64 spatial attention positions (8×8 grid) vs. variable number in Part I

### Step 5 — Decoder (Single-Layer LSTM)

```python
class DecoderRNN(nn.Module):
    def __init__(self, embed_size, vocab_size, attention_dim, encoder_dim, decoder_dim, drop_prob=0.3):
        self.lstm_cell = nn.LSTMCell(embed_size + encoder_dim, decoder_dim, bias=True)
        self.f_beta    = nn.Linear(decoder_dim, encoder_dim)     # defined but not applied in predict
        self.fcn       = nn.Linear(decoder_dim, vocab_size)
        self.drop      = nn.Dropout(drop_prob)
```

**Batch greedy decoding (generate_caption):**
```python
def generate_caption(self, features, max_len=200, itos=None, stoi=None):
    h, c      = self.init_hidden_state(features)
    word      = torch.full((batch_size, 1), stoi['<sos>']).to(device)
    embeds    = self.embedding(word)
    captions  = torch.zeros((batch_size, 202), dtype=torch.long).to(device)
    captions[:, 0] = word.squeeze()

    for i in range(202):                               # fixed 202 steps, no per-sample early stop
        alpha, context = self.attention(features, h)
        lstm_input = torch.cat((embeds[:, 0], context), dim=1)
        h, c = self.lstm_cell(lstm_input, (h, c))
        output = self.fcn(self.drop(h)).view(batch_size, -1)
        predicted_word_idx = output.argmax(dim=1)
        captions[:, i] = predicted_word_idx
        embeds = self.embedding(predicted_word_idx).unsqueeze(1)

    return captions    # (batch_size, 202)
```

## Key Design Choices

| Choice | Part I Rationale | Part II Rationale |
|---|---|---|
| EfficientNet-B5 vs. ResNet-18 | Larger, more accurate backbone; requires pretrained weights | Smaller, faster, fully trainable with lower memory cost |
| 2-layer LSTM vs. 1-layer | Deeper decoder for complex InChI token dependencies | Simpler baseline; single-layer sufficient for character-level vocab |
| Space-split tokenization vs. character-level | Larger meaningful tokens = shorter sequences; InChI sublayers are semantically coherent units | Char-level is simpler (no pre-fitting needed); 41 chars covers full InChI alphabet |
| Sigmoid gate (`f_beta`) in Part I | Learned gating controls how much spatial context to blend at each step | Omitted in Part II for simplicity |
| ReLU (Part I) vs. tanh (Part II) in attention | ReLU is asymmetric; works well with the ReLU-heavy EfficientNet features | tanh is zero-centered; pairs naturally with LSTM's tanh activations |
| Portrait fix (Part I only) | Albumentations-based orientation correction prevents aspect distortion on tall images | torchvision resize to square anyway; omitted for simplicity |
| Fixed 202 steps vs. early stopping (Part I) | Per-sample `<eos>` detection stops wasted computation once a sequence ends | Fixed-length simpler to implement; captions truncated at `<eos>` in post-processing |
| Full InChI generation (Part II) vs. prefix prepend (Part I) | Part I tokenizes after `"InChI=1S/"` — shorter sequences to learn | Part II char-level vocab includes all InChI chars; model must learn prefix too |

## Suggested Improvements

1. **Switch to beam search** — Part I has `forward_step` already implemented for beam search; adding a beam wrapper (k=5) reduces Levenshtein by ~0.3–0.5. Part II's fixed-step decode should be refactored to support per-sample early stopping first
2. **Unfreeze Part I encoder for fine-tuning** — Part I loads frozen EfficientNet-B5 weights; fine-tuning with a low encoder LR (1e-5) while training the decoder at higher LR (4e-4) is standard practice and improves quality
3. **Replace Part II's 8×8 grid with a richer spatial representation** — ResNet-18's 8×8 = 64 attention pixels is very coarse for molecular diagrams; using a larger input (384×384) or removing an additional stride layer from the backbone increases the spatial grid to 12×12 or 16×16
4. **Replace LSTM decoder with a Transformer decoder** — self-attention over previously generated tokens handles long-range InChI dependencies (ring closure digit matching) that LSTM struggles with; top solutions all use Transformer decoders
5. **Upgrade Part II to space-split tokenization** — character-level generation is harder to learn (model must learn `"InChI=1S/"` prefix repeatedly); space-split tokens from Part I produce shorter, more meaningful sequences
6. **Fix `debug=True` in Part I** — limits inference to 5,000 images; set `CFG.debug=False` for complete submission
7. **Apply InChI validity post-processing** — use RDKit to verify chemical validity of predicted strings; invalid predictions score maximum Levenshtein distance and can be replaced with a fallback
8. **Ensemble Part I and Part II predictions** — two models with different architectures and tokenization strategies produce complementary errors; selecting the lower-length-normalized Levenshtein prediction per image improves overall score
